[ DC PJ 분석방향 ] : 모듈화! 추상화!

-대상 사이트: https://www.among-mountains-lies-a-silicon-god.online/

1.분석의 기준
(1)기능별로 구성이 독립된 모듈로 분석함
ex)배너 스타일, 스와이프 메뉴 스타일, 컨텐츠 요약 소개 스타일, ...
-> bs, sms, sis, ... 등으로 모듈화 하여 별도의 명칭으로 로딩함

(2)기능별 구성의 데이터구조를 일치시키기 위한 분석
ex)배너 스타일이 다른 카테고리별로 내용만 다름 -> 데이터 구조가 같음.
=분류체계는 같고 내용만 다름

2.카테고리 구성하기
(1)내용별 구성보다는 유형별로 구성할 것
-> 아무개 소개하기 (X) 👉 요약 소개 모듈 (O)
(2)카테고리 구성 전개를 유형 구성으로 작성하기
ex)상동모+배모+요소모1+요소모2+스메모+동모+단링모+하동모
-상동모 : 상단동일모듈
-배모 : 배너모듈
-요소모 : 요약소개모듈
-스메모 : 스와이프메뉴모듈
-동모 : 동영상모듈
-단링모 : 단순링크모듈
-하동모 : 하단동일모듈

-각 모듈에는 타이틀이 기본적으로 들어가 있음
-html5 설계도에 article이나 section 태그에 제목태그가 들어가는 게 기본으로 되어 있듯이 제목은 다 기본적으로 들어가 있음

-모듈이 한눈에 다 들어와야 하니까 설계를 할 때 아예 모바일로만 설계하기
->모바일을 데스크톱으로 봤을 때 어정쩡한 부분만 수정하도록 하기

-폭이나 글씨크기는 적용할 때 조금씩 다르게 적용할 수도 있음

(3)카테고리별 모듈구성으로 작성 후 그 모듈에 대한 카테고리 데이터를 매칭하기
-같은 유형의 모듈에 실제로 들어올 데이터 구조에 맞는 실제 데이터를 표시함
ex)비디오 카테고리 상단에 '동모'가 들어온 상황
->데이터 구조 : 비디오 + 제목 + 설명
->비디오정보 + 제목내용 + 설명내용
->{vinfo : "", tit : "", desc : ""}

-데이터 구조 즉, 객체화 구조를 뽑아내기
-설계도에 실제로 들어가는 데이터는 표시하지 않아도 됨
=더미데이터로 표시해도 됨

-공통 모듈, 파트모듈로 나눠서 해당 모듈에 들어가는 구조를 표현하면 됨(?)
